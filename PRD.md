# PRD — 3D-концепт Куба прогресса на Three.js

## 1. Документ
**Название проекта:** Куб прогресса  
**Формат:** интерактивный веб-концепт  
**Цель:** быстро собрать и протестировать визуальный и интерактивный концепт куба для презентации клиенту и внутренней проверки идеи  
**Статус:** draft  
**Приоритет:** высокий

---

## 2. Контекст

Нужно собрать демонстрационный 3D-концепт куба, который станет центральным объектом лендинга кампании. Куб должен визуально передавать идею прогресса, вовлечения и сборки результата усилиями сотрудников.

Это не полноценный симулятор кубика Рубика и не production-версия механики. Это быстрый, красивый и понятный proof of concept, который позволит:
- проверить, как работает визуальная метафора куба
- показать клиенту hero-объект в живом виде
- понять, насколько убедительно выглядит вращение и поворот граней
- использовать концепт как основу для дальнейшей продуктовой реализации

---

## 3. Цель продукта

Создать браузерный интерактивный прототип 3D-куба на Three.js, который:
- выглядит как премиальный digital-объект
- медленно вращается вокруг своей оси
- позволяет по кнопке поворачивать выбранную грань
- даёт ощущение, что пользователь влияет на объект
- хорошо подходит для показа в презентации и на демо

---

## 4. Что именно нужно сделать

Нужно собрать интерактивную сцену с 3D-кубом:
- куб состоит из маленьких сегментов в логике 3x3x3
- куб медленно вращается в idle-состоянии
- у пользователя есть кнопки для поворота отдельных граней
- поворот грани анимируется плавно на 90 градусов
- после поворота общий idle rotation продолжается
- сцена выглядит чисто, современно и презентационно

---

## 5. Product outcome

На выходе должен получиться рабочий прототип, который можно:
- открыть локально в браузере
- показать клиенту как концепт
- использовать как базу для дальнейшей доработки
- встроить в будущий лендинг или адаптировать под hero-блок

---

## 6. Scope

### Входит в scope
- 3D-сцена на Three.js
- куб 3x3x3 из маленьких кубов
- базовые материалы и свет
- мягкое общее вращение куба
- кнопки для поворота 1–3 граней
- плавная анимация поворота на 90 градусов
- аккуратный минимальный UI
- адаптация под resize окна
- ограниченные OrbitControls
- чистая структура кода

### Не входит в scope
- полноценная физика реального Rubik’s Cube
- логика собранности куба
- подсчёт прогресса, коинов, заданий
- интеграция с Telegram
- backend
- сохранение состояния
- mobile production optimization
- полноценный лендинг вокруг сцены

---

## 7. Пользовательский сценарий

### Основной сценарий
1. Пользователь открывает страницу
2. Видит крупный красивый 3D-куб в центре экрана
3. Куб медленно вращается
4. Пользователь нажимает на кнопку поворота грани
5. Одна грань плавно поворачивается на 90 градусов
6. Куб продолжает общее вращение
7. Пользователь может повторять взаимодействие

### Дополнительный сценарий
1. Пользователь слегка вращает камеру мышкой
2. Видит куб с разных сторон
3. Камера остаётся в контролируемом диапазоне и не ломает композицию

---

## 8. UX-принципы

- Объект должен ощущаться как центральный hero-элемент
- Взаимодействие должно быть простым и мгновенно понятным
- Анимации должны быть плавными и дорогими по ощущению
- Интерфейс не должен спорить с кубом за внимание
- Весь концепт должен выглядеть как B2B tech и premium presentation object
- Пользователь должен чувствовать влияние на объект, даже при минимальном взаимодействии

---

## 9. Визуальные требования

### Общий стиль
- чистый
- современный
- премиальный
- технологичный
- презентационный
- не игровой и не мультяшный

### Куб
- структура 3x3x3
- аккуратные сегменты с небольшими зазорами
- тёмные разделители между сегментами
- слегка глянцевые материалы
- хорошие блики, но без дешёвого пластика
- ощущение корпоративного digital-объекта

### Цвета
Использовать палитру, близкую к концепту презентации:
- зелёный
- салатовый
- жёлтый
- светло-голубой
- синий
- белый или светло-серый

Цвета должны быть:
- чистыми
- современными
- не кислотными
- согласованными между собой

### Фон
- светлый
- нейтральный
- чистый
- без перегруза
- можно использовать очень мягкий градиент

### Свет
- мягкий studio-light setup
- читаемые объёмы
- лёгкая тень под объектом
- возможно слабое ощущение отражения или студийной поверхности

## 10. Функциональные требования

### Idle rotation
- куб медленно вращается вокруг своей оси
- вращение должно работать постоянно, пока не идёт активная анимация грани
- скорость должна быть комфортной и ненавязчивой

### Поворот грани
- по нажатию на кнопку должна поворачиваться выбранная грань
- поворот должен идти на 90 градусов
- анимация должна быть плавной
- во время поворота не должно быть конфликтов с повторным нажатием
- после завершения нужно корректно вернуть сегменты в основную структуру

### Кнопки управления
Минимально:
- Повернуть правую грань

Желательно:
- Повернуть правую грань
- Повернуть верхнюю грань
- Повернуть переднюю грань

### OrbitControls
- включить для демонстрации
- ограничить углы, чтобы пользователь не ломал красивый ракурс
- отключить слишком сильный zoom или сделать его ограниченным

### Resize
- сцена должна корректно адаптироваться под изменение размеров окна
- объект должен оставаться хорошо видимым

---

## 11. Технические требования

### Стек
- Three.js
- JavaScript
- HTML
- CSS

### Формат результата
Предпочтительно:
- `index.html`
- `style.css`
- `main.js`

Допустимо:
- один HTML-файл со встроенными стилями и скриптом для быстрого теста

### Структура
Код должен быть разделён логически:
- создание сцены
- создание камеры
- свет
- сборка куба
- логика idle-вращения
- логика поворота грани
- UI-события
- анимационный цикл

### Архитектура куба
- куб собирается из 27 маленьких кубиков
- для поворота выбранной грани используется временная группировка нужных кубиков
- после завершения поворота элементы возвращаются в общую иерархию

### Производительность
- проект должен работать плавно на обычном ноутбуке
- не использовать тяжёлые шейдеры и лишние эффекты
- сцена должна быть достаточно лёгкой для демо

---

## 12. UI

### Состав интерфейса
- минимальный заголовок или подпись
- блок кнопок управления
- сам куб в центре сцены

### Стиль UI
- современный
- минимальный
- лёгкий
- без перегруза
- в духе продуктового лендинга

### Расположение
- кнопки можно разместить снизу или справа
- UI не должен перекрывать куб
- должен сохраняться акцент на 3D-объекте

---

## 13. Параметры, которые должны легко настраиваться в коде

В коде должно быть явно понятно, где менять:
- цвета граней
- скорость idle-вращения
- скорость поворота грани
- размер куба
- свет
- стартовый ракурс камеры
- список доступных кнопок

---

## 14. Критерии приёмки

Прототип считается готовым, если:
1. Куб открывается локально без дополнительной сложной настройки
2. Куб визуально выглядит как качественный hero-объект
3. Есть мягкое автоматическое вращение
4. Есть минимум одна кнопка поворота грани
5. Поворот грани происходит плавно и понятно
6. После поворота сцена остаётся стабильной
7. Интерфейс не выглядит перегруженным
8. Код достаточно чистый, чтобы его можно было доработать дальше

---

## 15. Опциональные улучшения

Если останется время, можно добавить:
- лёгкое свечение или акцентный rim light
- hover-эффекты на кнопках
- мягкий post-processing без перегруза
- индикатор, какая грань будет вращаться
- плавное замедление и ускорение idle-вращения
- автопоказ нескольких поворотов в демо-режиме

---

## 16. Риски

- Если делать слишком реалистичную физику Rubik’s Cube, концепт резко усложнится
- Если дать пользователю слишком свободную камеру, может сломаться красивая композиция
- Если материалы будут слишком пластиковыми, объект потеряет premium-ощущение
- Если UI будет слишком активным, он начнёт отвлекать от куба

---

## 17. Definition of Done

Готовый результат:
- локально открывается
- выглядит красиво
- показывает идею куба как центрального объекта
- даёт интерактив через кнопки
- подходит для демонстрации клиенту и внутреннего теста концепции
